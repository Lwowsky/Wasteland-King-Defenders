# Технічна документація v130

## Адмінка — гравці

Файл: `js/pages/admin-page.js`

Таблиця `Зареєстровані гравці` працює через:

- `listRegisteredUsersPage()`;
- `ADMIN_PLAYERS_PAGE_SIZE = 10`;
- кнопки `Назад / Вперед`;
- основний гравець рахується у 10;
- ферми показуються під основою, якщо увімкнено `Основи + ферми`, але не рахуються як окремі 10.

## Firestore читання

Файл: `js/services/user-db.js`

Основний шлях:

1. Читає малу сторінку з `users`.
2. Сортує нових гравців за `createdAt`, якщо поле існує.
3. Фільтри застосовуються безпечно, щоб старі документи без `regionAccess` не пропадали.

Fallback:

1. Якщо індексований запит не повернув гравців, запускається малий legacy-запит.
2. Якщо і він не допоміг, використовується `publicPlayers` як індекс UID, після чого читаються тільки відповідні `users/{uid}` для редагування.
3. Повне читання всієї колекції не використовується.

## JSON-кеш

У v130 блок `Статистика з JSON-кешу` прибрано з адмінки. JSON-кеш може існувати для публічної статистики або інших сторінок, але адмінська таблиця гравців не повинна підміняти редагування гравців JSON-даними.

## Ліміти Firebase

Firebase вкладка показує локальну оцінку з `usage-tracker.js`, тому не потребує billing/card і не викликає Google Cloud Monitoring API.

Cloudflare вкладка залишена з реальними даними через Worker secret.
