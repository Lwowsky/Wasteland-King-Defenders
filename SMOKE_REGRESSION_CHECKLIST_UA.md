# Phase169 — Smoke / Regression checklist

Мета: швидко перевірити, що після модульного розбиття сайт працює у ключових flows.

## 1. Базове завантаження
- Відкрити `index.html`.
- Переконатися, що сторінка відкривається без білого екрану.
- Відкрити DevTools Console і перевірити, що немає `ReferenceError` / `SyntaxError` під час старту.
- Переключити мову UI, перевірити, що інтерфейс оновлюється.

## 2. Players / Table
- Додати кілька гравців вручну.
- Змінити ролі, tier, march-cap / параметри, якщо вони є у формі.
- Переконатися, що таблиця оновлюється без дублювань або втрати рядків.
- Оновити сторінку і перевірити, що локальний state відновлюється.

## 3. Import flow
- Завантажити файл через import.
- Перевірити auto-detect заголовків.
- Перевірити mapping колонок.
- Застосувати import і перевірити, що дані потрапили у таблицю.
- Перезавантажити сторінку і перевірити restore session.

## 4. Filters / Board
- Змінити top filters.
- Переконатися, що board / lists перемальовуються коректно.
- Перевірити live language sync на board.

## 5. Tower settings
- Відкрити налаштування турелі.
- Змінити параметри турелі.
- Зберегти і перевірити, що зміни застосовані на board.
- Закрити/відкрити модалку ще раз — стан має бути консистентним.

## 6. Assignments
- Призначити кількох гравців у турелі.
- Перевірити валідацію, конфлікти і статуси рядків.
- Видалити / перепризначити одного гравця.
- Переконатися, що assignments не ламають board render.

## 7. Autofill
- Запустити autofill.
- Перевірити, що гравці розкладаються без явних дублювань.
- Перевірити поведінку при нестачі доступних гравців.
- Повторно запустити autofill після ручних змін.

## 8. Tower calc modal
- Відкрити tower calc modal.
- Перевірити open/close.
- Змінити inline tower settings всередині modal.
- Перевірити reserve / overflow блок.
- Перевірити compute results / summary cards.
- Повторно відкрити modal і перевірити hydration стану.

## 9. Final plan
- Відкрити Final Plan вкладку/режим.
- Перевірити preview render.
- Перемкнути shift preview.
- Перевірити language layer у Final Plan.
- Натиснути apply до tower settings / towers.
- Переконатися, що після apply board не порожній.

## 10. Reset / patches / modal refresh
- Відкрити і закрити пов’язані модалки кілька разів.
- Перевірити reset confirm dialog.
- Перевірити reset columns / reset tables / full reset, якщо вони потрібні.
- Після reset перезавантажити сторінку і перевірити очікуваний чистий state.

## 11. Export / Share
- Перевірити export дії Final Plan / usage / інші доступні export actions.
- Якщо є screenshot/export board — перевірити, що не падає UI.

## 12. Критерій “готово”
Можна вважати збірку стабільною, якщо:
- немає стартових помилок у console;
- всі модалки відкриваються / закриваються;
- import, assignments, autofill, tower calc, final plan проходяться без блокуючих помилок;
- reload не руйнує state;
- Final Plan не зникає після open/apply/reopen.

## Якщо знаходиш баг
Для кожного бага варто зафіксувати:
- точний сценарій кроків;
- чи відтворюється після reload;
- текст помилки в console;
- скріншот;
- який саме архів перевірявся.
