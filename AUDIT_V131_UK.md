# AUDIT v131 — adminUsersIndex + cheap admin search

## Що змінено

- Додано приватну колекцію `adminUsersIndex/{uid}` для адмінської таблиці гравців.
- Адмінка більше не повинна шукати гравців через повну колекцію `users`.
- Пошук запускається тільки кнопкою **Оновити** або клавішею **Enter** у полі пошуку. Набір кожної букви більше не робить Firestore-запит.
- Додано кнопку **Оновити індекс**, яка один раз читає до 500 профілів і створює легкий індекс для дешевого пошуку.
- Редагування гравця лишилось через `users/{uid}`: список читає легкий індекс, а збереження оновлює повний профіль і індекс.
- Додано правила Firestore для `adminUsersIndex`: читати й писати можуть тільки Owner/Admin/Moderator.
- Виправлено помилку в `managerPublicPlayerSafe()` у `firestore.rules`.

## Перевірено

- `node --check` для JS-файлів — OK.
- `console.log` — не знайдено.
- `onSnapshot` — не знайдено.
- Дублі функцій у `admin-page.js` і `user-db.js` — не знайдено.
- Дублі нових i18n-ключів — не знайдено.

## Після деплою

1. Оновити `firebase/firestore.rules` у Firebase Console.
2. Задеплоїти Worker: `npx wrangler deploy`.
3. В адмінці натиснути **Оновити індекс** один раз.
4. Після цього пошук/фільтри мають працювати через `adminUsersIndex`.
