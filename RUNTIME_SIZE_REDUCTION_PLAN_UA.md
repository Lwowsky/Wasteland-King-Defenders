# Runtime size reduction plan

Поточний `release_runtime_only` уже близький до мінімального складу за файлами: усі локальні `js/*.js` реально підключені в `index.html`.

## Що ще можна зменшити далі

### 1. Production bundle
Найбільший реальний виграш дасть окремий production-етап:
- зібрати 1 `app.bundle.js` замість десятків окремих файлів;
- зібрати 1 `app.bundle.css`;
- лишити оригінальні модулі як source/debug пакет окремо.

Це безпечніше робити вже **після smoke/regression**.

### 2. HTML cleanup
`index.html` зараз один із найбільших runtime-файлів. Після стабілізації можна:
- винести повторювані inline-блоки в JS/CSS;
- прибрати мертві data-атрибути / зайві коментарі;
- скоротити дубльовану розмітку, якщо вона знайдеться.

### 3. i18n optimization
`js/i18n.js` — найбільший JS-файл.
Його не чіпали спеціально для безпеки. Пізніше можна окремим етапом:
- винести словники в окремі JSON/JS чанки;
- завантажувати мову lazy-load;
- прибрати дублікати ключів, якщо вони є.

### 4. Compat cleanup
Після повного smoke-прогону можна прибрати частину compat-шару, якщо підтвердиться, що він більше не потрібен.

## Що не варто робити зараз
- не робити ще одне масове ручне переписування модулів;
- не чіпати `i18n.js` до завершення regression pass;
- не видаляти compat-код до підтвердження, що всі flows живі.
