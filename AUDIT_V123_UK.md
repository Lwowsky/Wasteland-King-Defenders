# AUDIT v124 — адмінка, ліміти, безпека

## Що виправлено

1. Вкладка Адмінка → Гравці більше не зависає на порожній таблиці, якщо старі документи users не мають службових полів `createdAt`, `regionAccess` або `allianceAccess`.
2. Додано безпечний fallback-запит для старих документів: спочатку сайт пробує економний індексований запит, а якщо він повернув 0 гравців, читає маленьку порцію legacy-документів, а не всю колекцію.
3. Cloudflare вкладка тепер показує локальну оцінку використання:
   - Worker requests / day;
   - D1 rows read / day;
   - D1 rows written / day.
4. Додано кнопку скидання локальної оцінки Cloudflare.
5. Пояснення у вкладці Cloudflare оновлено: реальні Cloudflare-метрики не можна безпечно тягнути напряму з фронтенду, для цього потрібен Worker із secret API token.
6. Переклади оновлено для UA, EN, RU, PL, DE, JA, ZH, KO, VI, AR.

## Економія лімітів

Основний режим залишається економним:

- перша сторінка гравців: `limit(11)` для 10 гравців + перевірка наступної сторінки;
- ферми не рахуються окремими Firestore reads, бо вони лежать у документі користувача;
- fallback для старих документів читає максимум малу порцію, а не 4000+ документів.

## Безпека

У `firebase/firestore.rules` приватний документ `users/{uid}` з email читають тільки:

- власник профілю;
- Owner;
- Admin;
- Moderator.

Consul і Officer мають працювати з публічними даними `publicPlayers` або регіональними таблицями без email.
