Чистка

Очищення старих резервів без зміни активної D1-таблиці, архіву циклів і фінального плану.

Старі Firestore-заявки

Це старий Firebase-резерв. Активна D1-таблиця, D1-архів циклів і поточний cycleId не чіпаються. Заявки 14+ днів можна чистити пачками.

Перевірка не запускалась. Автоочистка запускається тільки для Admin, максимум раз на 24 години, і тільки для заявок старше 14 днів.

Контроль секретних посилань

Показує активні D1-посилання форми, таблиці і фінального плану для вибраного регіону. Можна перевипустити код без Firestore.

Перевірка секретних посилань ще не запускалась.

D1 зберігання

D1 storage росте від рядків активної таблиці, архівів циклів, фінальних планів, повідомлень/кампаній і службових індексів.

Очищення запускається тільки вручну й видаляє максимум 500 D1 рядків за раз.
Щоб зменшити D1 розмір: чисть старі цикли/фінальні плани/секретні посилання 14+ днів або старше. Активний цикл не видаляється.

Ліміти

Окрема вкладка тільки для адміна: Firebase, Cloudflare і ручне очищення старих архівів.

Ліміти Firebase

Орієнтовний локальний лічильник сайту + довідкові ліміти Firebase Spark.

Очищення Firebase архівів

Старі Firestore-повідомлення, кампанії та журнали після переносу в D1 можна чистити вручну порціями.

Очищення запускається тільки вручну й видаляє максимум 500 документів за раз.

Ліміти Cloudflare

Реальні дані Cloudflare через Worker secret + довідкові ліміти Workers Free і D1 Free.

Показані довідкові ліміти Cloudflare. Натисни Оновити, щоб отримати реальні дані й зберегти їх локально для економії запитів.

Ліміти GitHub

Repo size, останній deploy, GitHub Actions і Pages status через Worker secret.

Показані довідкові GitHub ліміти. Для реальних даних додай GITHUB_TOKEN, GITHUB_OWNER і GITHUB_REPO у Worker secrets.
